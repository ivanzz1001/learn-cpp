# C++将亡值

在 C++ 中，将亡值（eXpiring value，xvalue） 是 C++11 引入的一种**值类别**，它表示一个**具有身份（可以取地址）但即将被移动（资源可以被窃取） 的对象**。将亡值属于泛左值（glvalue） 的子集，同时也是右值（rvalue） 的子集，它是移动语义的核心概念之一。


## 1 值类别体系回顾（C++11 起）

C++ 表达式的值类别分为三个基本类别，并组合出两种复合类别：

```txt
       expression
       /        \
   glvalue     rvalue
    /    \     /    \
lvalue   xvalue   prvalue
```

- **左值（lvalue）**：有身份，不可移动（例如变量名、返回左值引用的函数调用）。

- **纯右值（prvalue）**：没有身份，用于初始化或运算（例如字面量 42、临时对象、返回非引用类型的函数调用）。

- **将亡值（xvalue）**：有身份，可移动。

- **泛左值（glvalue）** = 左值 + 将亡值（有身份）。

- **右值（rvalue）** = 将亡值 + 纯右值（可移动）。

将亡值同时具备“有身份”和“可移动”的特性，这使它成为连接传统左值与移动语义的桥梁。

## 2. 将亡值的核心特征

- **有身份**：可以取地址（`&xvalue` 合法），因为它仍然对应于某个对象的内存位置。

- **可移动** ：它的资源可以被安全地“窃取”，因为该对象即将销毁（或生命周期即将结束）。

- **属于右值**：因此可以绑定到右值引用（`T&&`），触发移动语义。


## 3. 将亡值的产生方式
将亡值通常由以下操作产生：

1) **使用 std::move**

std::move 将一个左值显式转换为将亡值（右值引用）：

```cpp
std::vector<int> v = {1,2,3};
std::vector<int>&& x = std::move(v); // x 绑定到将亡值（v 被转换为将亡值）
```

2) **返回右值引用的函数调用**

```cpp
int&& f();
auto&& y = f(); // f() 返回将亡值
```

3) **显式类型转换为右值引用**

```cpp
static_cast<std::vector<int>&&>(v);
```

4) **通过右值引用访问对象成员**

```cpp
struct S { int x; };
S s;
std::move(s).x; // 产生将亡值，表示 s.x 可以被移动
```

## 4. 将亡值与移动语义

将亡值是移动语义的“触发器”。当一个将亡值被用来初始化一个对象或赋值时，编译器会选择移动构造函数或移动赋值运算符（如果可用），从而高效地转移资源。

```cpp
std::vector<int> v1 = {1,2,3};
std::vector<int> v2 = std::move(v1); // std::move(v1) 产生将亡值，调用移动构造函数
```

这里 std::move(v1) 的结果是将亡值，它允许 v2 窃取 v1 的堆内存，而无需深拷贝。

## 5. 将亡值的生命周期
将亡值对应的对象仍然存在（它有身份），但其生命周期即将结束（或被移动）。移动操作后，原对象通常处于**“有效但未指定”**的状态，可以被重新赋值或安全销毁。

例如：

```cpp
std::string a = "hello";
std::string b = std::move(a); // a 现在为空（但标准只保证有效未指定）
a = "world";                   // a 可以重新使用
```

## 6. 将亡值与纯右值的区别

<div class="table-wrapper" markdown="block">
  
|    特性          |   将亡值(xvalue)                       | 纯右值(prvalue)              |
|:-----------------|:-------------------------------------:|:-----------------------------|
|是否有身份         | 是（可取地址）                         | 否（不能取地址）               |
|典型例子	        | `std::move(x)`、`static_cast<T&&>(x)`	| `42`、`T()`、`a+b`           |
|生命周期	        | 与源对象绑定，移动后源对象仍存在	       | 临时对象，表达式结束后销毁     |
|是否可移动         | 是	                                 | 是                          |
|是否属于泛左值     | 是	                                 | 否                          |
|是否属于右值	    | 是                                     | 是                          |
</div>

## 7. 将亡值的常见误区
**误区**：将亡值就是右值引用。

**事实**：将亡值是一种值类别（表达式的分类），而右值引用是一种类型（如 int&&）。右值引用变量本身是左值，而将亡值是由某些表达式（如 std::move(x)）产生的值类别。

<br />

**误区**：将亡值对象在移动后立即销毁。

**事实**：移动后原对象仍然存在（除非是临时对象），只是资源被转移。原对象的析构函数会在其生命周期结束时调用。

## 8. 总结
将亡值（xvalue）是 C++ 值类别中一个精巧的设计，它结合了“有身份”和“可移动”两个看似矛盾的属性，使得我们可以安全、高效地转移资源。理解将亡值有助于深入掌握移动语义、右值引用以及现代 C++ 中的资源管理。

**一句话记忆**：将亡值 = 有身份的右值，它代表“即将被移动的资源”
