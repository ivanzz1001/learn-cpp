## 1. C++左值详解

在 C++ 中，左值（lvalue） 是表达式的一种值类别（value category），它表示一个具有身份（identity）且不能被移动的对象（或位域）。简单来说，左值指向一个明确的内存位置，你可以取得它的地址，并且它在表达式结束后依然存在。

理解左值是掌握 C++ 基础的关键，特别是随着 C++11 引入移动语义和右值引用后，值分类变得更加精细，但左值的核心概念依然不变。

### 1.1 历史起源：为什么叫“左值”？

名字源自 C 语言：左值 是指可以出现在赋值运算符 左侧 的表达式。例如：

```pp
int x = 42;
x = 100;         // x 是左值，因为它可以出现在 = 左边
```

相应地，右值（rvalue） 最初指只能出现在赋值右侧的表达式，如字面量`42`或临时计算结果。

不过在现代 C++ 中，这种简单的左右划分已经不够用了，但名字保留了下来。

### 1.2 左值的核心特征
一个表达式是左值，当且仅当它具备以下特征:

1） **可以取地址:** 使用`&` 运算符能获得其内存地址

```cpp
int x = 10;
&x;               // 合法，x 是左值
// &42;           // 错误，42 是右值，不能取地址
```

2）**有持久的状态：** 左值通常对应一个变量、函数、数组元素、数据成员等，其生命周期由作用域决定

3）**可以出现在赋值左边**（除非被 const 修饰）

```cpp
const int c = 5;
// c = 10;        // 错误：c 是左值，但不可修改
```
4） **可以隐式转换为右值：** 当需要一个右值时，左值会自动进行左值到右值转换（例如作为算术运算符的操作数）


### 1.3 左值的常见例子

### 1.4 左值引用（lvalue reference）

C++ 使用 `&`声明左值引用，它只能绑定到左值（或某些特殊情况下的 const 左值可绑定到右值，但那是例外）。

```cpp
int x = 10;
int& ref = x;           // 合法，ref 绑定到左值 x
// int& ref2 = 20;      // 错误：20 是右值，不能绑定到左值引用
const int& ref3 = 30;   // 合法：const 左值引用可以延长右值的生命周期
```
左值引用常用于避免拷贝、实现输出参数等。

### 1.5 现代C++值类别体系(C++11起)

C++11引入了移动语义，值类别被重新定义为核心的三类：**左值（lvalue）**、**纯右值（prvalue）**、**将亡值（xvalue）**，并由此组合出**泛左值（glvalue）**和**右值（rvalue）**。

```txt
       expression
       /        \
   glvalue     rvalue
    /    \     /    \
lvalue   xvalue   prvalue
```

- **左值(lvalue)**: 具有身份，且不可移动(仍未传统左值)

- **将亡值(xvalue)**: 具有身份，但可以被移动（例如 std::move() 返回的值、某些类型转换的结果）

- **纯右值(prvalue)**: 没有身份，用于初始化或运算（如临时对象、字面量）

- **泛左值（glvalue）**  = 左值 + 将亡值（有身份）。

- **右值（rvalue）** = 将亡值 + 纯右值（可移动）。

在这个体系中，左值依然是核心，但将亡值也拥有身份（可取地址），却被视为右值引用，可以被移动。

### 1.6 如何判断一个表达式是左值？

- **使用** `decltype`：若 `decltype(expr)` 返回引用类型（如 `int&`），则 `expr` 是左值（或将亡值，但通常左值居多）。若返回非引用，则可能是纯右值。

- **尝试取地址**：若 `&expr` 合法，则 `expr` 是左值（或将亡值，但将亡值取地址也是合法的）。

- **标准库类型萃取**：std::is_lvalue_reference、std::is_rvalue_reference 等可以判断引用类型，但不能直接判断值类别。

- **常见的左值产生方式**：

  - 变量名、函数名（函数类型是左值？注意函数本身不是数据对象，但函数指示符在某些语境是左值）

  - 返回左值引用的函数调用

  - 内置赋值、复合赋值、前置自增/自减运算符的结果

  - 字符串字面量

  - 成员访问（如果成员本身是左值）

### 1.7 左值到右值转换

当需要一个右值（如进行算术运算）而提供了一个左值时，编译器自动执行 左值到右值转换，读取左值的内容产生一个纯右值。

```cpp
int x = 10;
int y = x + 5;   // x 是左值，但加法需要右值，所以发生左值到右值转换，读取 x 的值 10
```

这种转换不会改变左值本身，只是提取它的值

### 1.8 左值在移动语义中的角色

移动语义主要针对右值（尤其是将亡值），但左值也可以显式转换为右值使用 std::move():

```cpp
std::vector<int> v1 = {1,2,3};
std::vector<int> v2 = std::move(v1);  // v1 被转换为将亡值，资源被移动
// 移动后 v1 处于有效但未指定状态
```

这里 std::move(v1) 产生一个将亡值，它有身份（仍可取地址），但被当作右值绑定到移动构造函数，从而窃取资源。

### 1.9 总结

- **左值** 是代表内存位置的表达式，可以取地址、有持久身份。

- 它可以是变量、数组元素、解引用的指针、返回左值引用的函数调用等。

- 左值引用 `T&` 绑定到左值，const 左值引用可绑定到右值以延长生命周期。

- 现代 C++ 中，左值与将亡值、纯右值共同构成精细的值类别体系，但左值仍是其中最基本的一类。

- 理解左值是理解引用、移动语义、完美转发等高级特性的基石。

掌握左值的概念，能帮助你写出更高效、更正确的 C++ 代码，尤其是在涉及引用和移动操作时。


## 2. C++ 右值详解

在 C++ 中，**右值（rvalue）** 是表达式的一种**值类别（value category）**，它表示一个**没有持久身份、即将被销毁或临时存在的值**。最初，右值指只能出现在赋值运算符右侧的表达式，但随着 C++11 引入移动语义，右值的概念被细分为纯右值（prvalue）和将亡值（xvalue），但核心思想仍然是：右值代表“可以安全移动的资源”。

理解右值是掌握现代 C++（尤其是移动语义和完美转发）的基础。

### 2.1 历史起源：为什么叫“右值”？

名字源自 C 语言：右值 是指只能出现在赋值运算符 右侧 的表达式。例如:

```cpp
int x = 42;   // 42 是右值，只能出现在 = 右边
x = x + 1;    // x+1 的结果是右值
```
相比之下，左值可以出现在左侧（如变量 x）。这个命名至今仍有参考意义，但现代 C++ 的值类别更加复杂。

### 2.2 右值的核心特征

一个表达式是右值，当且仅当它具备以下特征：

1) **不能取地址**：使用 `&` 运算符无法获得其内存地址（除非是将亡值，但将亡值也属于右值，且可以取地址——这需要区分）

```cpp
// &42;               // 错误：42 是纯右值，不能取地址
// &(x + 1);          // 错误：x+1 是纯右值
int&& rref = 42;
// &rref;             // 合法：rref 是变量名，本身是左值
```

2) **没有持久身份**：右值通常是临时对象、字面量或表达式计算过程中产生的中间值，它们在表达式结束后就会被销毁。

3) **可以出现在赋值右侧**：任何右值都可以被用来初始化变量或赋值给左值。

4) **可以绑定到右值引用**：右值可以被右值引用（T&&）捕获，从而延长其生命周期或实现移动语义

### 2.3 右值的分类（C++11 起）

C++11 将右值进一步划分为两类：

#### 纯右值（prvalue, pure rvalue）

- **定义**：没有身份（无法取地址）的临时对象、字面量或求值结果。

- **例子**：

  - 字面量：42、true、nullptr

  - 临时对象：std::string("hello")

  - 返回非引用类型的函数调用：std::to_string(123)

  - 算术表达式：a + b、x++（后置自增）

  - 枚举项

- **特点**：纯右值用于初始化对象或作为操作数，生命周期通常很短。

#### 将亡值（xvalue, eXpiring value）

- **定义**：有身份（可取地址），但即将被移动（资源可被窃取）的值。它将资源所有权转移给另一个对象。

- **例子：**

  - std::move(x) 的返回值

  - 返回右值引用的函数调用：std::move(x)、static_cast<int&&>(x)

  - 通过右值引用访问的对象成员（如 std::move(obj).member）

- **特点**：将亡值可以被移动，同时也保留了身份（可以取地址），因此属于泛左值（glvalue）的子集，同时也是右值的子集。

### 2.4 右值引用（rvalue reference）

C++11 引入 `T&&` 语法，用于声明**右值引用**。右值引用只能绑定到右值（包括纯右值和将亡值），但不能直接绑定到左值（除非用 std::move 转换）。

```cpp
int&& rref1 = 42;            // 绑定到纯右值 42
int x = 10;
int&& rref2 = std::move(x);  // 绑定到将亡值（x 被转换为右值）
// int&& rref3 = x;          // 错误：不能将左值绑定到右值引用
```

#### 右值引用的作用

1) **延长临时对象生命周期**：与 const 左值引用类似，右值引用可以让临时对象存活到引用作用域结束。

```cpp
std::vector<int>&& rref = std::vector<int>{1,2,3}; // 临时对象生命周期延长
```

2) **实现移动语义**：函数重载可以区分左值和右值，对于右值可以窃取其资源（如动态内存），避免深拷贝。

```cpp
class MyString {
public:
    // 移动构造函数
    MyString(MyString&& other) noexcept
        : data(other.data), size(other.size) {
        other.data = nullptr;  // 窃取资源，将源对象置为空
    }
};
```

3) **完美转发**：在模板中结合 std::forward，可以保持参数的原始值类别（左值或右值）传递给其他函数。

### 2.5 右值在移动语义中的角色
移动语义允许将资源（如堆内存、文件句柄）从一个对象转移到另一个对象，而不是拷贝。这主要针对右值，因为右值即将被销毁，窃取其资源是安全的。

- **std::move**：一个类型转换工具，将左值转换为将亡值（右值引用），从而触发移动操作

  ```cpp
  std::vector<int> v1 = {1,2,3};
  std::vector<int> v2 = std::move(v1);  // v1 被移动，v2 获得其资源
  // 移动后 v1 处于有效但未指定状态（通常为空）
  ```

- **移动构造函数/移动赋值运算符**：接受 `T&&` 参数，实现资源转移

### 2.6 如何判断一个表达式是右值？

- **使用**`decltype`：

  - 如果 `decltype(expr)` 是非引用类型（如 `int`），则 `expr` 是纯右值。

  - 如果 `decltype(expr)` 是右值引用类型（如 `int&&`），则 `expr` 是将亡值（属于右值）。

  - 如果 `decltype(expr)` 是左值引用（如 `int&`），则 `expr` 是左值。

- **尝试取地址：**

  - 如果 `&expr` 非法，则 `expr` 通常是纯右值（将亡值可取地址，需结合其他判断）。

- **标准库类型萃取：**

  - `std::is_rvalue_reference<T>::value` 判断类型是否为右值引用，但不能直接判断表达式的值类别。

  - `std::is_lvalue_reference`、`std::is_rvalue_reference` 用于引用类型。

- **常见右值表达式**：

  - 字面量（字符串字面量除外，它是左值）

  - 临时对象（如`ClassName()`）

  - 后置自增/自减：`x++`、`x--`

  - 算术、逻辑、比较表达式：`a + b`、`a && b`、`a == b`

  - 返回非引用类型的函数调用：`func()`

  - 转型为右值引用：`static_cast<T&&>(obj)`、`std::move(obj)`

### 2.7 右值到左值的转换

右值本身不能直接当左值用，但可以通过以下方式变成左值：

- **绑定到左值引用**：`const T& `可以绑定到右值，延长其生命周期，此时右值通过引用变成了左值（因为引用变量名是左值）

  ```cpp
  const int& ref = 42;   // 42 是右值，ref 是左值引用，ref 本身是左值
  ```

- **绑定到右值引用**：右值引用变量本身是左值（因为它有名字和地址）

  ```cpp
  int&& rref = 42;       // rref 是左值（变量名）
  // 需要使用 std::move(rref) 再次转换为右值才能移动
  ```

### 2.8 右值引用与完美转发

完美转发允许函数模板将其参数原样（保持左值/右值属性）传递给另一个函数：

```cpp
template<typename T>
void wrapper(T&& arg) {           // 通用引用（转发引用）
    foo(std::forward<T>(arg));    // 完美转发
}
```

- 当传入左值时，`T` 推导为 `T&`，`std::forward` 保持左值引用。

- 当传入右值时，`T` 推导为 `T`，`std::forward` 转换为右值引用

这是利用右值引用和引用折叠实现的。

### 2.9 常见误区澄清
**误区**：右值引用变量本身是右值。

**事实**：右值引用变量是左值（有名字，可取地址）。要将其作为右值传递，需用 std::move。

<br />

**误区**：字符串字面量 "hello" 是右值。

**事实**：字符串字面量是左值，它存储在静态内存区，有持久身份。

<br />

**误区**：所有临时对象都是右值。

**事实**：临时对象通常是纯右值，但可以通过引用绑定变成左值（如 `const T&` 延长生命周期后，引用本身是左值）。

<br />

**误区**：移动语义会自动发生。

**事实**：移动需要显式触发（如 std::move）或函数重载匹配右值，编译器不会自动将左值当作右值移动。
